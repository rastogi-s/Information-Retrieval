   <h1 class="firstHeading" id="firstHeading" lang="en">
    Collation
   </h1>
   <div class="mw-body-content" id="bodyContent">
    <div class="noprint" id="siteSub">

    </div>

    </div>

     <div class="mw-parser-output">
      <div class="hatnote navigation-not-searchable" role="note">
       This article is about collation in library, information, and computer science. For other uses, see
       <a class="mw-disambig" href="/wiki/Collation_(disambiguation)" title="Collation (disambiguation)">
        Collation (disambiguation)
       </a>
       .
      </div>
      <p>
       <b>
        Collation
       </b>
       is the assembly of written information into a standard order. Many systems of collation are based on
       <a href="/wiki/Number" title="Number">
        numerical order
       </a>
       or
       <a href="/wiki/Alphabetical_order" title="Alphabetical order">
        alphabetical order
       </a>
       , or extensions and combinations thereof. Collation is a fundamental element of most office
       <a href="/wiki/Library_classification" title="Library classification">
        filing systems
       </a>
       ,
       <a href="/wiki/Library_catalog" title="Library catalog">
        library catalogs
       </a>
       , and
       <a class="mw-redirect" href="/wiki/Reference_book" title="Reference book">
        reference books
       </a>
       .
      </p>
      <p>
       Collation differs from
       <i>
        <a href="/wiki/Document_classification" title="Document classification">
         classification
        </a>
       </i>
       in that classification is concerned with arranging information into logical categories, while collation is concerned with the ordering of items of information, usually based on the form of their
       <a href="/wiki/Identifier" title="Identifier">
        identifiers
       </a>
       . Formally speaking, a collation method typically defines a
       <a href="/wiki/Total_order" title="Total order">
        total order
       </a>
       on a set of possible identifiers, called
       <a class="mw-redirect" href="/wiki/Sort_key" title="Sort key">
        sort keys
       </a>
       , which consequently produces a
       <a class="mw-redirect" href="/wiki/Total_preorder" title="Total preorder">
        total preorder
       </a>
       on the set of items of information (items with the same identifier are not placed in any defined order).
      </p>
      <p>
       A collation algorithm such as the
       <a href="/wiki/Unicode_collation_algorithm" title="Unicode collation algorithm">
        Unicode collation algorithm
       </a>
       defines an order through the process of comparing two given
       <a class="mw-redirect" href="/wiki/Character_string" title="Character string">
        character strings
       </a>
       and deciding which should come before the other. When an order has been defined in this way, a
       <i>
        <a href="/wiki/Sorting_algorithm" title="Sorting algorithm">
         sorting algorithm
        </a>
       </i>
       can be used to put a list of any number of items into that order.
      </p>
      <p>
       The main advantage of collation is that it makes it fast and easy for a user to find an element in the list, or to confirm that it is absent from the list. In automatic systems this can be done using a
       <a href="/wiki/Binary_search_algorithm" title="Binary search algorithm">
        binary search algorithm
       </a>
       or
       <a href="/wiki/Interpolation_search" title="Interpolation search">
        interpolation search
       </a>
       ; manual searching may be performed using a roughly similar procedure, though this will often be done unconsciously. Other advantages are that one can easily find the first or last elements on the list (most likely to be useful in the case of numerically sorted data), or elements in a given range (useful again in the case of numerical data, and also with alphabetically ordered data when one may be sure of only the first few letters of the sought item or items).
      </p>
      <p>
      </p>

      <h2>
       <span class="mw-headline" id="Numerical_and_chronological_order">
        Numerical and chronological order
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=1" title="Edit section: Numerical and chronological order">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       Strings representing
       <a href="/wiki/Number" title="Number">
        numbers
       </a>
       may be sorted based on the values of the numbers that they represent. For example, "−4", "2.5", "10", "89", "30,000". Note that pure application of this method may provide only a partial ordering on the strings, since different strings can represent the same number (as with "2" and "2.0" or, when
       <a href="/wiki/Scientific_notation" title="Scientific notation">
        scientific notation
       </a>
       is used, "2e3" and "2000").
      </p>
      <p>
       A similar approach may be taken with strings representing
       <a href="/wiki/Calendar_date" title="Calendar date">
        dates
       </a>
       or other items that can be ordered chronologically or in some other natural fashion.
      </p>
      <h2>
       <span class="mw-headline" id="Alphabetical_order">
        Alphabetical order
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=2" title="Edit section: Alphabetical order">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/Alphabetical_order" title="Alphabetical order">
        Alphabetical order
       </a>
      </div>
      <p>
       <a href="/wiki/Alphabetical_order" title="Alphabetical order">
        Alphabetical order
       </a>
       is the basis for many systems of collation where items of information are identified by strings consisting principally of
       <a href="/wiki/Letter_(alphabet)" title="Letter (alphabet)">
        letters
       </a>
       from an
       <a href="/wiki/Alphabet" title="Alphabet">
        alphabet
       </a>
       . The ordering of the strings relies on the existence of a standard ordering for the letters of the alphabet in question. (The system is not limited to alphabets in the strict technical sense; languages that use a
       <a href="/wiki/Syllabary" title="Syllabary">
        syllabary
       </a>
       or
       <a href="/wiki/Abugida" title="Abugida">
        abugida
       </a>
       , for example
       <a href="/wiki/Cherokee_language" title="Cherokee language">
        Cherokee
       </a>
       , can use the same ordering principle provided there is a set ordering for the symbols used.)
      </p>
      <p>
       To decide which of two strings comes first in alphabetical order, initially their first letters are compared. The string whose first letter appears earlier in the alphabet comes first in alphabetical order. If the first letters are the same, then the second letters are compared, and so on, until the order is decided. (If one string runs out of letters to compare, then it is deemed to come first; for example, "cart" comes before "carthorse".) The result of arranging a set of strings in alphabetical order is that words with the same first letter are grouped together, and within such a group words with the same first two letters are grouped together, and so on.
      </p>
      <p>
       <a class="mw-redirect" href="/wiki/Capital_letter" title="Capital letter">
        Capital letters
       </a>
       are typically treated as equivalent to their corresponding lowercase letters. (For alternative treatments in computerized systems, see
       <a href="#Automated_collation">
        Automated collation
       </a>
       , below.)
      </p>
      <p>
       Certain limitations, complications, and special conventions may apply when alphabetical order is used:
      </p>
      <ul>
       <li>
        When strings contain
        <a class="mw-redirect" href="/wiki/Space_(character)" title="Space (character)">
         spaces
        </a>
        or other word dividers, the decision must be taken whether to ignore these dividers or to treat them as symbols preceding all other letters of the alphabet. For example, if the first approach is taken then "car park" will come after "carbon" and "carp" (as it would if it were written "carpark"), whereas in the second approach "car park" will come before those two words. The first rule is used in many (but not all)
        <a href="/wiki/Dictionary" title="Dictionary">
         dictionaries
        </a>
        , the second in
        <a href="/wiki/Telephone_directory" title="Telephone directory">
         telephone directories
        </a>
        (so that Wilson, Jim K appears with other people named Wilson, Jim and not after Wilson, Jimbo).
       </li>
       <li>
        Abbreviations may be treated as if they were spelt out in full. For example, names containing "St." (short for the English word
        <i>
         <a href="/wiki/Saint" title="Saint">
          Saint
         </a>
        </i>
        ) are often ordered as if they were written out as "Saint". There is also a traditional convention in English that surnames beginning
        <i>
         Mc
        </i>
        and
        <i>
         M'
        </i>
        are listed as if those prefixes were written
        <i>
         Mac
        </i>
        .
       </li>
       <li>
        Strings that represent personal names will often be listed by alphabetical order of surname, even if the
        <a href="/wiki/Given_name" title="Given name">
         given name
        </a>
        comes first. For example, Juan Hernandes and Brian O'Leary should be sorted as "Hernandes, Juan" and "O'Leary, Brian" even if they are not written this way.
       </li>
       <li>
        Very common initial words, such as
        <i>
         The
        </i>
        in English, are often ignored for sorting purposes. So
        <i>
         <a href="/wiki/The_Shining_(novel)" title="The Shining (novel)">
          The Shining
         </a>
        </i>
        would be sorted as just "Shining" or "Shining, The".
       </li>
       <li>
        When some of the strings contain
        <a href="/wiki/Numerical_digit" title="Numerical digit">
         numerals
        </a>
        (or other non-letter characters), various approaches are possible. Sometimes such characters are treated as if they came before or after all the letters of the alphabet. Another method is for numbers to be sorted alphabetically as they would be spelled: for example
        <i>
         <a href="/wiki/1776_(film)" title="1776 (film)">
          1776
         </a>
        </i>
        would be sorted as if spelled out "seventeen seventy-six", and
        <i>
         <a class="mw-redirect" href="/wiki/24_heures_du_Mans" title="24 heures du Mans">
          24 heures du Mans
         </a>
        </i>
        as if spelled "vingt-quatre..." (French for "twenty-four"). When numerals or other symbols are used as special graphical forms of letters, as in
        <i>
         1337
        </i>
        for
        <a href="/wiki/Leet" title="Leet">
         leet
        </a>
        or
        <i>
         Se7en
        </i>
        for the movie title
        <i>
         <a href="/wiki/Seven_(1995_film)" title="Seven (1995 film)">
          Seven
         </a>
        </i>
        , they may be sorted as if they were those letters.
       </li>
       <li>
        Languages have different conventions for treating
        <a class="mw-redirect" href="/wiki/Modified_letter" title="Modified letter">
         modified letters
        </a>
        and certain letter combinations. For example, in
        <a href="/wiki/Spanish_language" title="Spanish language">
         Spanish
        </a>
        the letter
        <i>
         ñ
        </i>
        is treated as a basic letter following
        <i>
         n
        </i>
        , and the
        <a href="/wiki/Digraph_(orthography)" title="Digraph (orthography)">
         digraphs
        </a>
        <i>
         ch
        </i>
        and
        <i>
         ll
        </i>
        were formerly (until 1994) treated as basic letters following
        <i>
         c
        </i>
        and
        <i>
         l
        </i>
        , although they are now alphabetized as two-letter combinations. A list of such conventions for various languages can be found at
        <a href="/wiki/Alphabetical_order#Language-specific_conventions" title="Alphabetical order">
         Alphabetical order § Language-specific conventions
        </a>
        .
       </li>
      </ul>
      <p>
       In several languages the rules have changed over time, and so older dictionaries may use a different order than modern ones. Furthermore, collation may depend on use. For example, German
       <a href="/wiki/Dictionary" title="Dictionary">
        dictionaries
       </a>
       and
       <a href="/wiki/Telephone_directory" title="Telephone directory">
        telephone directories
       </a>
       use different approaches.
      </p>
      <h2>
       <span class="mw-headline" id="Radical-and-stroke_sorting">
        Radical-and-stroke sorting
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=3" title="Edit section: Radical-and-stroke sorting">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <dl>
       <dd>
        <i>
         See also
         <a href="/wiki/Chinese_characters#Indexing" title="Chinese characters">
          Indexing of Chinese characters
         </a>
        </i>
       </dd>
      </dl>
      <p>
       Another form of collation is
       <b>
        radical-and-stroke sorting
       </b>
       , used for non-alphabetic writing systems such as the
       <a class="mw-redirect" href="/wiki/Hanzi" title="Hanzi">
        hanzi
       </a>
       of
       <a href="/wiki/Chinese_language" title="Chinese language">
        Chinese
       </a>
       and the
       <a href="/wiki/Kanji" title="Kanji">
        kanji
       </a>
       of
       <a href="/wiki/Japanese_language" title="Japanese language">
        Japanese
       </a>
       , whose thousands of symbols defy ordering by convention. In this system, common components of characters are identified; these are called
       <a class="mw-redirect" href="/wiki/Radical_(Chinese_character)" title="Radical (Chinese character)">
        radicals
       </a>
       in Chinese and logographic systems derived from Chinese. Characters are then grouped by their primary radical, then ordered by number of pen strokes within radicals. When there is no obvious radical or more than one radical, convention governs which is used for collation. For example, the Chinese character 妈 (meaning "mother") is sorted as a six-stroke character under the three-stroke primary radical 女.
      </p>
      <p>
       The radical-and-stroke system is cumbersome compared to an alphabetical system in which there are a few characters, all unambiguous. The choice of which components of a logograph comprise separate radicals and which radical is primary is not clear-cut. As a result, logographic languages often supplement radical-and-stroke ordering with alphabetic sorting of a phonetic conversion of the logographs. For example, the kanji word
       <i>
        <a href="/wiki/Tokyo" title="Tokyo">
         Tōkyō
        </a>
       </i>
       (東京) can be sorted as if it were spelled out in the Japanese characters of the
       <a href="/wiki/Hiragana" title="Hiragana">
        hiragana
       </a>
       syllabary as "to-u-ki-
       <sub>
        yo
       </sub>
       -u" (とうきょう), using the conventional sorting order for these characters.
       <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
        [
        <i>
         <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
          <span title="This claim needs references to reliable sources. (October 2012)">
           citation needed
          </span>
         </a>
        </i>
        ]
       </sup>
      </p>
      <p>
       In addition, in Greater China,
       <a href="/wiki/Surname_stroke_order" title="Surname stroke order">
        surname stroke ordering
       </a>
       is a convention in some official documents where people's names are listed without hierarchy.
      </p>
      <p>
       The radical-and-stroke system, or some similar pattern-matching and stroke-counting method, was traditionally the only practical method for constructing dictionaries that someone could use to look up a logograph whose pronunciation was unknown. With the advent of computers, dictionary programs are now available that allow one to handwrite a character using a mouse or stylus.
       <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
        [
        <i>
         <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
          <span title="This claim needs references to reliable sources. (October 2012)">
           citation needed
          </span>
         </a>
        </i>
        ]
       </sup>
      </p>
      <h2>
       <span class="mw-headline" id="Automated_collation">
        Automated collation
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=4" title="Edit section: Automated collation">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       When information is stored in digital systems, collation may become an automated process. It is then necessary to implement an appropriate collation
       <a href="/wiki/Algorithm" title="Algorithm">
        algorithm
       </a>
       that allows the information to be sorted in a satisfactory manner for the application in question. Often the aim will be to achieve an alphabetical or numerical ordering that follows the standard criteria as described in the preceding sections. However, not all of these criteria are easy to automate.
       <sup class="reference" id="cite_ref-Walters_1-0">
        <a href="#cite_note-Walters-1">
         [1]
        </a>
       </sup>
      </p>
      <p>
       The simplest kind of automated collation is based on the numerical codes of the symbols in a
       <a class="mw-redirect" href="/wiki/Character_set" title="Character set">
        character set
       </a>
       , such as
       <a href="/wiki/ASCII" title="ASCII">
        ASCII
       </a>
       coding (or any of its
       <a class="mw-redirect" href="/wiki/Superset" title="Superset">
        supersets
       </a>
       such as
       <a href="/wiki/Unicode" title="Unicode">
        Unicode
       </a>
       ), with the symbols being ordered in increasing numerical order of their codes, and this ordering being extended to strings in accordance with the basic principles of alphabetical ordering (mathematically speaking,
       <a href="/wiki/Lexicographical_order" title="Lexicographical order">
        lexicographical ordering
       </a>
       ). So a computer program might treat the characters
       <i>
        a
       </i>
       ,
       <i>
        b
       </i>
       ,
       <i>
        C
       </i>
       ,
       <i>
        d
       </i>
       , and
       <i>
        $
       </i>
       as being ordered
       <i>
        $
       </i>
       ,
       <i>
        C
       </i>
       ,
       <i>
        a
       </i>
       ,
       <i>
        b
       </i>
       ,
       <i>
        d
       </i>
       (the corresponding ASCII codes are
       <i>
        $
       </i>
       = 36,
       <i>
        a
       </i>
       = 97,
       <i>
        b
       </i>
       = 98,
       <i>
        C
       </i>
       = 67, and
       <i>
        d
       </i>
       = 100). Therefore, strings beginning with
       <i>
        C
       </i>
       ,
       <i>
        M
       </i>
       , or
       <i>
        Z
       </i>
       would be sorted before strings with lower-case
       <i>
        a
       </i>
       ,
       <i>
        b
       </i>
       , etc. This is sometimes called
       <i>
        <a class="mw-redirect" href="/wiki/ASCIIbetical_order" title="ASCIIbetical order">
         ASCIIbetical order
        </a>
       </i>
       . This deviates from the standard alphabetical order, particularly due to the ordering of capital letters before all lower-case ones (and possibly the treatment of spaces and other non-letter characters). It is therefore often applied with certain alterations, the most obvious being case conversion (often to uppercase, for historical reasons
       <sup class="reference" id="cite_ref-2">
        <a href="#cite_note-2">
         [note 1]
        </a>
       </sup>
       ) before comparison of ASCII values.
      </p>
      <p>
       In many collation algorithms, the comparison is based not on the numerical codes of the characters, but with reference to the
       <b>
        collating sequence
       </b>
       – a sequence in which the characters are assumed to come for the purpose of collation – as well as other ordering rules appropriate to the given application. This can serve to apply the correct conventions used for alphabetical ordering in the language in question, dealing properly with differently cased letters,
       <a class="mw-redirect" href="/wiki/Modified_letter" title="Modified letter">
        modified letters
       </a>
       ,
       <a href="/wiki/Digraph_(orthography)" title="Digraph (orthography)">
        digraphs
       </a>
       , particular abbreviations, and so on, as mentioned above under
       <a href="#Alphabetical_order">
        Alphabetical order
       </a>
       , and in detail in the
       <a href="/wiki/Alphabetical_order" title="Alphabetical order">
        Alphabetical order
       </a>
       article. Such algorithms are potentially quite complex, possibly requiring several passes through the text.
       <sup class="reference" id="cite_ref-Walters_1-1">
        <a href="#cite_note-Walters-1">
         [1]
        </a>
       </sup>
      </p>
      <p>
       Problems are nonetheless still common when the algorithm has to encompass more than one language. For example, in
       <a class="mw-redirect" href="/wiki/German_(language)" title="German (language)">
        German
       </a>
       dictionaries the word
       <i>
        ökonomisch
       </i>
       comes between
       <i>
        offenbar
       </i>
       and
       <i>
        olfaktorisch
       </i>
       , while
       <a class="mw-redirect" href="/wiki/Turkish_(language)" title="Turkish (language)">
        Turkish
       </a>
       dictionaries treat
       <i>
        o
       </i>
       and
       <i>
        ö
       </i>
       as different letters, placing
       <i>
        oyun
       </i>
       before
       <i>
        öbür
       </i>
       .
      </p>
      <p>
       A standard algorithm for collating any collection of strings composed of any standard
       <a href="/wiki/Unicode" title="Unicode">
        Unicode
       </a>
       symbols is the
       <a class="mw-redirect" href="/wiki/Unicode_Collation_Algorithm" title="Unicode Collation Algorithm">
        Unicode Collation Algorithm
       </a>
       . This can be adapted to use the appropriate collation sequence for a given language by tailoring its default collation table. Several such tailorings are collected in
       <a href="/wiki/Common_Locale_Data_Repository" title="Common Locale Data Repository">
        Common Locale Data Repository
       </a>
       .
      </p>
      <h3>
       <span class="mw-headline" id="Sort_keys">
        Sort keys
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=5" title="Edit section: Sort keys">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h3>
      <div class="hatnote navigation-not-searchable" role="note">
       "Sort key" redirects here. For sort keys in Wikipedia, see
       <a class="mw-redirect" href="/wiki/Wikipedia:SORTKEY" title="Wikipedia:SORTKEY">
        WP:SORTKEY
       </a>
       .
      </div>
      <p>
       In some applications, the strings by which items are collated may differ from the identifiers that are displayed. For example,
       <i>
        The Shining
       </i>
       might be sorted as
       <i>
        Shining, The
       </i>
       (see
       <a href="#Alphabetical_order">
        Alphabetical order
       </a>
       above), but it may still be desired to display it as
       <i>
        The Shining
       </i>
       . In this case two sets of strings can be stored, one for display purposes, and another for collation purposes. Strings used for collation in this way are called
       <i>
        sort keys
       </i>
       .
      </p>
      <h3>
       <span class="mw-headline" id="Issues_with_numbers">
        Issues with numbers
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=6" title="Edit section: Issues with numbers">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h3>
      <p>
       Sometimes, it is desired to order text with embedded numbers using proper numerical order. For example, "Figure 7b" goes before "Figure 11a", even though '7' comes after '1' in
       <a href="/wiki/Unicode" title="Unicode">
        Unicode
       </a>
       . This can be extended to
       <a class="mw-redirect" href="/wiki/Roman_numeral" title="Roman numeral">
        Roman numerals
       </a>
       . This behavior is not particularly difficult to produce as long as only integers are to be sorted, although it can slow down sorting significantly. For example,
       <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">
        Microsoft Windows
       </a>
       does this when sorting
       <a class="mw-redirect" href="/wiki/File_name" title="File name">
        file names
       </a>
       .
      </p>
      <p>
       Sorting decimals properly is a bit more difficult, because different locales use different symbols for a
       <a href="/wiki/Decimal_separator" title="Decimal separator">
        decimal point
       </a>
       , and sometimes the same character used as a
       <a class="mw-redirect" href="/wiki/Decimal_mark" title="Decimal mark">
        decimal point
       </a>
       is also used as a separator, for example "Section 3.2.5". There is no universal answer for how to sort such strings; any rules are application dependent.
      </p>
      <p>
       Ascending order of numbers differs from alphabetical order, e.g. 11 comes alphabetically before 2. This can be fixed with
       <a href="/wiki/Leading_zero" title="Leading zero">
        leading zeros
       </a>
       : 02 comes alphabetically before 11. See e.g.
       <a href="/wiki/ISO_8601" title="ISO 8601">
        ISO 8601
       </a>
       .
      </p>
      <p>
       Also −13 comes alphabetically after −12 although it is less. With negative numbers, to make ascending order correspond with alphabetical sorting, more drastic measures are needed such as adding a constant to all numbers to make them all positive.
      </p>
      <h2>
       <span class="mw-headline" id="Labeling_of_ordered_items">
        Labeling of ordered items
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Collation&amp;action=edit&amp;section=7" title="Edit section: Labeling of ordered items">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       In some contexts, numbers and letters are used not so much as a basis for establishing an ordering, but as a means of labeling items that are already ordered. For example, pages, sections, chapters, and the like, as well as the items of lists, are frequently "numbered" in this way. Labeling series that may be used include ordinary
       <a href="/wiki/Arabic_numerals" title="Arabic numerals">
        Arabic numerals
       </a>
       (1, 2, 3, ...),
       <a href="/wiki/Roman_numerals" title="Roman numerals">
        Roman numerals
       </a>
       (I, II, III, ... or i, ii, iii, ...), or letters (A, B, C, ... or a, b, c, ...). (An alternative method for indicating list items, without numbering them, is to use a
       <a class="mw-redirect" href="/wiki/Bulleted_list" title="Bulleted list">
        bulleted list
       </a>
       .)
      </p>
      <p>
       When letters of an alphabet are used for this purpose of
       <a href="/wiki/Enumeration" title="Enumeration">
        enumeration
       </a>
       , there are certain language-specific conventions as to which letters are used. For example, the
       <a href="/wiki/Russian_alphabet" title="Russian alphabet">
        Russian
       </a>
       letters
       <a class="mw-redirect" href="/wiki/%D0%AA" title="Ъ">
        Ъ
       </a>
       and
       <a class="mw-redirect" href="/wiki/%D0%AC" title="Ь">
        Ь
       </a>
       (which in writing are only used for modifying the preceding
       <a href="/wiki/Consonant" title="Consonant">
        consonant
       </a>
       ), and usually also
       <a class="mw-redirect" href="/wiki/%D0%AB" title="Ы">
        Ы
       </a>
       ,
       <a class="mw-redirect" href="/wiki/%D0%99" title="Й">
        Й
       </a>
       , and
       <a class="mw-redirect" href="/wiki/%D0%81" title="Ё">
        Ё
       </a>
       , are usually omitted. Also in many languages that use extended
       <a href="/wiki/Latin_script" title="Latin script">
        Latin script
       </a>
       , the
       <a class="mw-redirect" href="/wiki/Modified_letter" title="Modified letter">
        modified letters
       </a>
       are often not used in enumeration.
      </p>
