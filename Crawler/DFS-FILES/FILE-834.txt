      <p>
       <b>
        SWF
       </b>
       (
       <span class="nowrap">
        <span class="IPA nopopups noexcerpt">
         <a href="/wiki/Help:IPA/English" title="Help:IPA/English">
          /
          <span style="border-bottom:1px dotted">
           <span title="/ˈ/: primary stress follows">
            ˈ
           </span>
           <span title="'s' in 'sigh'">
            s
           </span>
           <span title="'w' in 'wind'">
            w
           </span>
           <span title="/ɪ/: 'i' in 'kit'">
            ɪ
           </span>
           <span title="'f' in 'find'">
            f
           </span>
          </span>
          /
         </a>
        </span>
       </span>
       <a href="/wiki/Help:Pronunciation_respelling_key" title="Help:Pronunciation respelling key">
        <i title="English pronunciation respelling">
         <span style="font-size:90%">
          SWIF
         </span>
        </i>
       </a>
       )
       <sup class="reference" id="cite_ref-3">
        <a href="#cite_note-3">
         [3]
        </a>
       </sup>
       is an abbreviation for
       <i>
        small web format
       </i>
       , an
       <a href="/wiki/Adobe_Flash" title="Adobe Flash">
        Adobe Flash
       </a>
       <a href="/wiki/File_format" title="File format">
        file format
       </a>
       used for
       <a href="/wiki/Multimedia" title="Multimedia">
        multimedia
       </a>
       ,
       <a href="/wiki/Vector_graphics" title="Vector graphics">
        vector graphics
       </a>
       and
       <a href="/wiki/ActionScript" title="ActionScript">
        ActionScript
       </a>
       .
       <sup class="reference" id="cite_ref-swfopen_4-0">
        <a href="#cite_note-swfopen-4">
         [4]
        </a>
       </sup>
       Originating with
       <a href="/wiki/FutureWave_Software" title="FutureWave Software">
        FutureWave Software
       </a>
       , then transferred to
       <a href="/wiki/Macromedia" title="Macromedia">
        Macromedia
       </a>
       , and then coming under the control of
       <a href="/wiki/Adobe_Systems" title="Adobe Systems">
        Adobe
       </a>
       , SWF files can contain
       <a href="/wiki/Animation" title="Animation">
        animations
       </a>
       or
       <a href="/wiki/Applet" title="Applet">
        applets
       </a>
       of varying degrees of interactivity and function. They may also occur in programs, commonly
       <a href="/wiki/Browser_game" title="Browser game">
        browser games
       </a>
       , using
       <a href="/wiki/ActionScript" title="ActionScript">
        ActionScript
       </a>
       .
      </p>
      <p>
       Programmers can generate SWF files from within several Adobe products, including
       <a href="/wiki/Adobe_Flash" title="Adobe Flash">
        Flash
       </a>
       ,
       <a href="/wiki/Adobe_Flash_Builder" title="Adobe Flash Builder">
        Flash Builder
       </a>
       (an
       <a href="/wiki/Integrated_development_environment" title="Integrated development environment">
        IDE
       </a>
       ),
       <a href="/wiki/Adobe_Animate" title="Adobe Animate">
        Adobe Animate
       </a>
       (the replacement for Adobe Flash as of Feb. 2016), and
       <a href="/wiki/Adobe_After_Effects" title="Adobe After Effects">
        After Effects
       </a>
       , as well as through MXMLC, a command-line application compiler which forms part of the freely-available
       <a class="mw-redirect" href="/wiki/Adobe_Flex" title="Adobe Flex">
        Flex SDK
       </a>
       . Although Adobe
       <a href="/wiki/Adobe_Illustrator" title="Adobe Illustrator">
        Illustrator
       </a>
       can generate SWF format files through its "export" function, it cannot open or edit them. Other than using Adobe products, one can build SWFs with
       <a class="mw-redirect" href="/wiki/Open-source" title="Open-source">
        open-source
       </a>
       <a href="/wiki/MTASC" title="MTASC">
        Motion-Twin ActionScript 2 Compiler (MTASC)
       </a>
       , the open-source
       <a href="/wiki/Ming_library" title="Ming library">
        Ming library
       </a>
       and the
       <a class="mw-redirect" href="/wiki/Free-software" title="Free-software">
        free-software
       </a>
       suite
       <a href="/wiki/SWFTools" title="SWFTools">
        SWFTools
       </a>
       . Various other third-party programs can also produce files in this format, such as
       <a class="mw-redirect" href="/wiki/Multimedia_Fusion_2" title="Multimedia Fusion 2">
        Multimedia Fusion 2
       </a>
       ,
       <a class="mw-redirect" href="/wiki/Captivate" title="Captivate">
        Captivate
       </a>
       and
       <a href="/wiki/SWiSH_Max" title="SWiSH Max">
        SWiSH Max
       </a>
       .
      </p>
      <p>
       The term "SWF" has originated as an abbreviation for
       <i>
        ShockWave Flash
       </i>
       .
       <sup class="reference" id="cite_ref-Schaeffer2007_5-0">
        <a href="#cite_note-Schaeffer2007-5">
         [5]
        </a>
       </sup>
       This usage was changed
       <sup class="noprint Inline-Template" style="white-space:nowrap;">
        [
        <i>
         <a href="/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions" title="Wikipedia:Manual of Style/Words to watch">
          <span title="The material near this tag may use weasel words or too-vague attribution. (May 2016)">
           by whom?
          </span>
         </a>
        </i>
        ]
       </sup>
       to the
       <a href="/wiki/Backronym" title="Backronym">
        backronym
       </a>
       <i>
        Small Web Format
       </i>
       to eliminate confusion with a different technology,
       <a href="/wiki/Adobe_Shockwave" title="Adobe Shockwave">
        Shockwave
       </a>
       , from which SWF derived.
       <sup class="reference" id="cite_ref-6">
        <a href="#cite_note-6">
         [6]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-NiederstRobbins2001_7-0">
        <a href="#cite_note-NiederstRobbins2001-7">
         [7]
        </a>
       </sup>
       There is no official resolution to the acronym "SWF" by Adobe.
       <sup class="reference" id="cite_ref-8">
        <a href="#cite_note-8">
         [8]
        </a>
       </sup>
      </p>

      <h2>
       <span class="mw-headline" id="History">
        History
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=SWF&amp;action=edit&amp;section=1" title="Edit section: History">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       The small company
       <a href="/wiki/FutureWave_Software" title="FutureWave Software">
        FutureWave Software
       </a>
       originally defined the file format with one primary objective: to create small files for displaying entertaining animations.
       <sup class="reference" id="cite_ref-9">
        <a href="#cite_note-9">
         [9]
        </a>
       </sup>
       The idea involved a format which player software could run on any system and which would work with slower network connections. FutureWave released
       <a class="mw-redirect" href="/wiki/FutureSplash_Animator" title="FutureSplash Animator">
        FutureSplash Animator
       </a>
       in May 1996. In December 1996 Macromedia acquired FutureWave and FutureSplash Animator became
       <a class="mw-redirect" href="/wiki/Macromedia_Flash" title="Macromedia Flash">
        Macromedia Flash
       </a>
       1.0.
      </p>
      <p>
       The original naming of SWF came out of Macromedia's desire to capitalize on the well-known
       <a class="mw-redirect" href="/wiki/Macromedia_Shockwave" title="Macromedia Shockwave">
        Macromedia Shockwave
       </a>
       brand;
       <a class="mw-redirect" href="/wiki/Macromedia_Director" title="Macromedia Director">
        Macromedia Director
       </a>
       produced Shockwave files for the end user, so the files created by their newer Flash product tried to capitalize on the already established brand. As Flash became more popular than Shockwave itself, this branding decision became more of a liability, so the format started to be referred to as simply SWF.
       <sup class="reference" id="cite_ref-Schaeffer2007_5-1">
        <a href="#cite_note-Schaeffer2007-5">
         [5]
        </a>
       </sup>
      </p>
      <p>
       Adobe acquired Macromedia in 2005.
       <sup class="reference" id="cite_ref-10">
        <a href="#cite_note-10">
         [10]
        </a>
       </sup>
      </p>
      <p>
       On May 1, 2008, Adobe dropped its licensing restrictions on the SWF format specifications, as part of the
       <a class="mw-redirect" href="/wiki/Open_Screen_Project" title="Open Screen Project">
        Open Screen Project
       </a>
       . However,
       <a href="/wiki/Rob_Savoye" title="Rob Savoye">
        Rob Savoye
       </a>
       , a member of the
       <a href="/wiki/Gnash_(software)" title="Gnash (software)">
        Gnash
       </a>
       development team, has pointed to some parts of the Flash format which remain closed.
       <sup class="reference" id="cite_ref-stillpartlyclosed_11-0">
        <a href="#cite_note-stillpartlyclosed-11">
         [11]
        </a>
       </sup>
       On July 1, 2008, Adobe released code to Google and Yahoo, which allowed their search engines to crawl and index SWF files.
       <sup class="reference" id="cite_ref-12">
        <a href="#cite_note-12">
         [12]
        </a>
       </sup>
      </p>

       <span class="mw-editsection">
        <span class="mw-editsection-bracket">

        </span>
        <a href="/w/index.php?title=SWF&amp;action=edit&amp;section=2" title="Edit section: Description">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       The main graphical primitive in SWF is the path, which is a chain of segments of primitive types, ranging from lines to
       <a href="/wiki/Spline_(mathematics)" title="Spline (mathematics)">
        splines
       </a>
       or
       <a class="mw-redirect" href="/wiki/Bezier_curve" title="Bezier curve">
        bezier curves
       </a>
       . Additional primitives like rectangles, ellipses, and even text can be built from these. The graphical elements in SWF are thus fairly similar to
       <a class="mw-redirect" href="/wiki/SVG" title="SVG">
        SVG
       </a>
       and
       <a class="mw-redirect" href="/wiki/MPEG-4_BIFS" title="MPEG-4 BIFS">
        MPEG-4 BIFS
       </a>
       . SWF also uses
       <a href="/wiki/Display_list" title="Display list">
        display lists
       </a>
       and allows naming and reusing previously defined components.
       <sup class="reference" id="cite_ref-compar_13-0">
        <a href="#cite_note-compar-13">
         [13]
        </a>
       </sup>
      </p>
      <p>
       The binary stream format SWF uses is fairly similar to
       <a href="/wiki/QuickTime" title="QuickTime">
        QuickTime
       </a>
       atoms, with a tag, length and payload—an organization that makes it very easy for (older) players to skip contents they don't support.
       <sup class="reference" id="cite_ref-compar_13-1">
        <a href="#cite_note-compar-13">
         [13]
        </a>
       </sup>
      </p>
      <p>
       Originally limited to presenting vector-based objects and images in a simple sequential manner, the format in its later versions allows audio (since Flash 3) and video (since Flash 6).
      </p>
      <p>
       Adobe introduced a new, low-level 3D API in version 11 of the Flash Player. Initially codenamed
       <a href="/wiki/Molehill" title="Molehill">
        Molehill
       </a>
       , the official name given to this API was ultimately
       <a href="/wiki/Stage3D" title="Stage3D">
        Stage3D
       </a>
       . It was intended to be an equivalent of
       <a href="/wiki/OpenGL" title="OpenGL">
        OpenGL
       </a>
       or
       <a href="/wiki/Direct3D" title="Direct3D">
        Direct3D
       </a>
       .
       <sup class="reference" id="cite_ref-Kaitila2011_14-0">
        <a href="#cite_note-Kaitila2011-14">
         [14]
        </a>
       </sup>
       <sup class="reference" style="white-space:nowrap;">
        :
        <span>
         9
        </span>
       </sup>
       In Stage3D
       <a href="/wiki/Shader" title="Shader">
        shaders
       </a>
       are expressed in a low-level language called
       <a href="/wiki/Stage3D#Features" title="Stage3D">
        Adobe Graphics Assembly Language
       </a>
       (AGAL).
       <sup class="reference" id="cite_ref-Kaitila2011_14-1">
        <a href="#cite_note-Kaitila2011-14">
         [14]
        </a>
       </sup>
       <sup class="reference" style="white-space:nowrap;">
        :
        <span>
         57
        </span>
       </sup>
       <sup class="reference" id="cite_ref-15">
        <a href="#cite_note-15">
         [15]
        </a>
       </sup>
      </p>
      <h2>
       <span class="mw-headline" id="Adoption">

      <p>

       <a href="/wiki/Adobe_Flash_Player" title="Adobe Flash Player">

       </a>

        web browsers
       </a>

       <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">
        Microsoft Windows
       </a>
       ,
       <a class="mw-redirect" href="/wiki/Mac_OS_X" title="Mac OS X">
        Mac OS X
       </a>

       <a href="/wiki/Linux" title="Linux">
        Linux
       </a>

       <a class="mw-redirect" href="/wiki/X86_architecture" title="X86 architecture">
        x86 architecture
       </a>

       <a href="/wiki/ARM_architecture" title="ARM architecture">
        ARM architecture
       </a>
       (
       <a class="mw-redirect" href="/wiki/Google_Chrome_OS" title="Google Chrome OS">
        Google Chrome OS
       </a>

      </p>
      <p>
       <a href="/wiki/GNU" title="GNU">
        GNU
       </a>

       <a href="/wiki/Free_software" title="Free software">
        free software
       </a>

       <a href="/wiki/Gnash_(software)" title="Gnash (software)">
        Gnash
       </a>
       under the
       <a href="/wiki/GNU_General_Public_License" title="GNU General Public License">
        GNU General Public License
       </a>
       (GPL). Despite being a declared high-priority GNU project, funding for Gnash was fairly limited.
       <sup class="reference" id="cite_ref-lwn_16-0">
        <a href="#cite_note-lwn-16">
         [16]
        </a>
       </sup>

       <a class="mw-redirect" href="/wiki/GNU_LGPL" title="GNU LGPL">
        LGPL
       </a>

       <a href="/wiki/Swfdec" title="Swfdec">
        Swfdec
       </a>
       .
       <a href="/wiki/Lightspark" title="Lightspark">
        Lightspark
       </a>

       <sup class="reference" id="cite_ref-lwn_16-1">
        <a href="#cite_note-lwn-16">
         [16]
        </a>
       </sup>
      </p>

       <a href="/wiki/Adobe_Shockwave" title="Adobe Shockwave">

       <sup class="reference" id="cite_ref-17">
        <a href="#cite_note-17">
         [17]
        </a>
       </sup>
       <a class="mw-redirect" href="/wiki/InDesign" title="InDesign">
        InDesign
       </a>
       CS6 can also produce some limited forms of SWF animations directly.
       <sup class="reference" id="cite_ref-SmithTeam2012_18-0">
        <a href="#cite_note-SmithTeam2012-18">
         [18]
        </a>
       </sup>
      </p>
      <p>
       Sony
       <a href="/wiki/PlayStation_Portable" title="PlayStation Portable">
        PlayStation Portable
       </a>
       consoles can play limited SWF files in Sony's web browser, beginning with firmware version 2.71. Both the
       <a href="/wiki/Nintendo" title="Nintendo">
        Nintendo
       </a>
       <a href="/wiki/Wii" title="Wii">
        Wii
       </a>
       <sup class="reference" id="cite_ref-19">
        <a href="#cite_note-19">
         [19]
        </a>

       <a href="/wiki/Sony" title="Sony">

       </a>

       <sup class="reference" id="cite_ref-20">
        <a href="#cite_note-20">
         [20]
        </a>
       </sup>
       consoles can run SWF files through their Internet browsers.
      </p>
      <p>
       <a class="mw-redirect" href="/wiki/Scaleform" title="Scaleform">
        Scaleform
       </a>
       GFx is a commercial alternative SWF player that features full
       <a href="/wiki/Hardware_acceleration" title="Hardware acceleration">
        hardware acceleration
       </a>
       using the
       <a href="/wiki/Graphics_processing_unit" title="Graphics processing unit">
        GPU
       </a>
       and has high conformance up to Flash 8 and AS2. Scaleform GFx is licensed as a
       <a class="mw-redirect" href="/wiki/Game_middleware" title="Game middleware">
        game middleware
       </a>
       solution and used by many PC and console 3D games for user interfaces,
       <a href="/wiki/HUD_(video_gaming)" title="HUD (video gaming)">
        HUDs
       </a>
       ,
       <a href="/wiki/Minigame" title="Minigame">
        mini games
       </a>
       , and video playback.
       <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
        [
        <i>
         <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
          <span title="This claim needs references to reliable sources. (August 2014)">
           citation needed
          </span>
         </a>
        </i>
        ]
       </sup>
      </p>
      <p>
       The newer 3D features of SWF have been seen as an alternative to
       <a href="/wiki/WebGL" title="WebGL">
        WebGL
       </a>
       , with a spurt of 3D engines like
       <a href="/wiki/Papervision3D" title="Papervision3D">
        Papervision3D
       </a>
       ,
       <sup class="reference" id="cite_ref-KerrKeats2009_21-0">
        <a href="#cite_note-KerrKeats2009-21">
         [21]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-Lively2010_22-0">
        <a href="#cite_note-Lively2010-22">
         [22]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-TondeurWinder2009_23-0">
        <a href="#cite_note-TondeurWinder2009-23">
         [23]
        </a>
       </sup>
       <a href="/wiki/Away3D" title="Away3D">
        Away3D
       </a>
       ,
       <sup class="reference" id="cite_ref-KerrKeats2009_21-1">
        <a href="#cite_note-KerrKeats2009-21">
         [21]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-Casperson2011_24-0">
        <a href="#cite_note-Casperson2011-24">
         [24]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-Ivanov2011_25-0">
        <a href="#cite_note-Ivanov2011-25">
         [25]
        </a>
       </sup>

       <sup class="reference" id="cite_ref-KerrKeats2009_21-2">

      <h2>
       <span class="mw-headline" id="Published_specifications">
        Published specifications
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=SWF&amp;action=edit&amp;section=4" title="Edit section: Published specifications">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       Adobe makes available a partial specification of SWF,
       <sup class="reference" id="cite_ref-30">
        <a href="#cite_note-30">
         [30]
        </a>
       </sup>
       most recently updated in January 2013 to reflect changes in SWF version 19.
       <sup class="reference" id="cite_ref-31">
        <a href="#cite_note-31">
         [31]
        </a>
       </sup>
       SWF versions have been decoupled from Flash player versions after Flash 10. Afterwards the version number of the SWF progressed rapidly; SWF version 19 corresponds to the new features added in Flash Player 11.6.
       <sup class="reference" id="cite_ref-32">
        <a href="#cite_note-32">
         [32]
        </a>
       </sup>
       Flash Player 14 uses SWF version 25.
       <sup class="reference" id="cite_ref-33">
        <a href="#cite_note-33">
         [33]
        </a>
       </sup>
      </p>
      <p>
       In 2008, the specifications document was criticized by
       <a href="/wiki/Rob_Savoye" title="Rob Savoye">
        Rob Savoye
       </a>
       , the lead developer of the
       <a href="/wiki/Gnash_(software)" title="Gnash (software)">
        Gnash
       </a>
       project, as missing "huge amounts" of information needed to completely implement SWF, omitting specifications for
       <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol" title="Real Time Messaging Protocol">
        RTMP
       </a>
       and
       <a class="mw-redirect" href="/wiki/Sorenson_codec" title="Sorenson codec">
        Sorenson Spark
       </a>
       .
       <sup class="reference" id="cite_ref-stillpartlyclosed_11-1">
        <a href="#cite_note-stillpartlyclosed-11">
         [11]
        </a>
       </sup>
       The
       <a class="mw-redirect" href="/wiki/Real_Time_Messaging_Protocol" title="Real Time Messaging Protocol">
        RTMP
       </a>
       specification
       <sup class="reference" id="cite_ref-34">
        <a href="#cite_note-34">
         [34]
        </a>
       </sup>
       was released publicly in June 2009.
       <sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">
        [
        <i>
         <a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed">
          <span title="This claim needs references to reliable sources. (August 2014)">
           citation needed
          </span>
         </a>
        </i>
        ]
       </sup>
       The Sorenson Spark codec is not Adobe's property.
      </p>
      <h2>
       <span class="mw-headline" id="Licensing">
        Licensing
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=SWF&amp;action=edit&amp;section=5" title="Edit section: Licensing">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       Until May 1, 2008, implementing software that plays SWF was disallowed by the specification's license.
       <sup class="reference" id="cite_ref-35">
        <a href="#cite_note-35">
         [35]
        </a>
       </sup>
       On that date, as part of its Open Screen Project, Adobe dropped all such restrictions on the SWF and
       <a href="/wiki/Flash_Video" title="Flash Video">
        FLV
       </a>
       formats.
       <sup class="reference" id="cite_ref-36">
        <a href="#cite_note-36">
         [36]
        </a>
       </sup>
      </p>
      <p>
       Implementing software which creates SWF files has always been permitted, on the condition that the resulting files
       <a href="/wiki/Rendering_(computer_graphics)" title="Rendering (computer graphics)">
        render
       </a>
       "error free in the latest publicly available version of Adobe Flash Player."
       <sup class="reference" id="cite_ref-37">
        <a href="#cite_note-37">
         [37]
        </a>
       </sup>
      </p>

        </th>
        <td>
         .f4v files are similar to iTunes M4V files which are based on
         <a class="mw-redirect" href="/wiki/MP4" title="MP4">
          MP4
         </a>
         and can be played back by Flash Player 9 Update 3 and above. F4V file format is second container format for Flash video and it differs from FLV file format. It is based on the
         <a href="/wiki/ISO_base_media_file_format" title="ISO base media file format">
          ISO base media file format
         </a>
         .
         <sup class="reference" id="cite_ref-FLV-F4V_42-0">
          <a href="#cite_note-FLV-F4V-42">
           [42]
          </a>
         </sup>
         <sup class="reference" id="cite_ref-newformats_43-0">
          <a href="#cite_note-newformats-43">
           [43]
          </a>
         </sup>
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .f4p
         </b>
        </th>
        <td>
         .f4p files are an Adobe suffix for media encrypted with the Adobe Access
         <a href="/wiki/Digital_rights_management" title="Digital rights management">
          digital rights management
         </a>
         scheme which is based on the same protection scheme that their RTMP protocol uses.
         <sup class="reference" id="cite_ref-newformats_43-1">
          <a href="#cite_note-newformats-43">
           [43]
          </a>
         </sup>
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .f4a
         </b>
        </th>
        <td>
         .f4a files are an Adobe suffix for iTunes M4A files that contain only audio streams.
         <sup class="reference" id="cite_ref-newformats_43-2">
          <a href="#cite_note-newformats-43">
           [43]
          </a>
         </sup>
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .f4b
         </b>
        </th>
        <td>
         .f4b files are an Adobe suffix for iTunes M4B
         <a href="/wiki/Audiobook" title="Audiobook">
          audiobook
         </a>
         files.
         <sup class="reference" id="cite_ref-newformats_43-3">
          <a href="#cite_note-newformats-43">
           [43]
          </a>
         </sup>
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .f4m
         </b>
        </th>
        <td>
         .f4m files are XML manifest files. Containing
         <a href="/wiki/Base64" title="Base64">
          base64
         </a>
         FLV onMetaData headers for an Adobe version of bit rate control
         <a href="/wiki/HTTP_Live_Streaming" title="HTTP Live Streaming">
          HTTP Live Streaming
         </a>
         .
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .f4f
         </b>
        </th>
        <td>
         .f4f files are MP4 atom-ized fragmented files. Containing
         <a href="/wiki/Flash_Video#Packets" title="Flash Video">
          FLV packets
         </a>
         .
        </td>
       </tr>
       <tr>
        <th>
         <b>
          <a href="/wiki/Adobe_SWC_file" title="Adobe SWC file">
           .swc
          </a>
         </b>
        </th>
        <td>
         .swc files are used for distributing components; they contain a compiled clip, the component's ActionScript class file, and other files that describe the component.
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .jsfl
         </b>
        </th>
        <td>
         .jsfl files are used to add functionality in the Flash Authoring environment; they contain JavaScript code and access the Flash JavaScript API.
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .swt
         </b>
        </th>
        <td>
         .swt files are 'templatized' forms of .swf files, used by Macromedia Generator
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .flp
         </b>
        </th>
        <td>
         .flp files are XML files used to reference all the document files contained in a Flash Project. Flash Projects allow the user to group multiple, related files together to assist in Flash project organization, compilation and build.
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .spl
         </b>
        </th>
        <td>
         .spl files are
         <a class="mw-redirect" href="/wiki/FutureSplash_Animator" title="FutureSplash Animator">
          FutureSplash Animator
         </a>
         documents.
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .aso
         </b>
        </th>
        <td>
         .aso files are cache files used during Flash development, containing compiled ActionScript byte code. An ASO file is recreated when a change in its corresponding class files is detected. Occasionally the Flash IDE does not recognize that a recompile is necessary, and these cache files must be deleted manually. They are located in %USERPROFILE%\Local Settings\Application Data\Macromedia\Flash8\en\Configuration\Classes\aso on Win32 / Flash8.
        </td>
       </tr>
       <tr>
        <th>
         <b>
          .sol
         </b>
        </th>
        <td>
         .sol files are created by
         <a href="/wiki/Adobe_Flash_Player" title="Adobe Flash Player">
          Adobe Flash Player
         </a>
         to hold
         <a class="mw-redirect" href="/wiki/Local_Shared_Object" title="Local Shared Object">
          Local Shared Objects
         </a>
         (data stored on the system running the Flash player).
        </td>
       </tr>
      </table>
