      <p>
       A
       <b>
        position representation
       </b>
       is the parameters used to express a position relative to a reference. Representing position in three dimensions is often done by a
       <a href="/wiki/Euclidean_vector" title="Euclidean vector">
        Euclidean vector
       </a>
       . However, when representing position relative to the
       <a href="/wiki/Earth" title="Earth">
        Earth
       </a>
       it is often more convenient to represent vertical position as
       <a href="/wiki/Altitude" title="Altitude">
        altitude
       </a>
       or depth, and to use some other parameters to represent
       <b>
        horizontal position
       </b>
       . There are also several applications where only the horizontal position is of interest, this might e.g. be the case for ships and ground vehicles/cars.
      </p>
      <p>
       There are several options for horizontal position representations, each with different properties which makes them appropriate for different applications.
       <a href="/wiki/Latitude" title="Latitude">
        Latitude
       </a>
       /
       <a href="/wiki/Longitude" title="Longitude">
        longitude
       </a>
       and
       <a href="/wiki/Universal_Transverse_Mercator_coordinate_system" title="Universal Transverse Mercator coordinate system">
        UTM
       </a>
       are common horizontal position representations.
      </p>
      <p>
       The horizontal position has two
       <a href="/wiki/Degrees_of_freedom_(mechanics)" title="Degrees of freedom (mechanics)">
        degrees of freedom
       </a>
       , and thus two parameters are sufficient to uniquely describe such a position. However, similarly to the use of
       <a href="/wiki/Euler_angles" title="Euler angles">
        Euler angles
       </a>
       as a
       <a href="/wiki/Rotation_formalisms_in_three_dimensions" title="Rotation formalisms in three dimensions">
        formalism for representing rotations
       </a>
       , using only the minimum number of parameters gives
       <a class="mw-redirect" href="/wiki/Mathematical_singularity" title="Mathematical singularity">
        singularities
       </a>
       , and thus three parameters are required for the horizontal position to avoid this.
      </p>
      <p>
      </p>

      <h2>
       <span class="mw-headline" id="Latitude_and_longitude">
        Latitude and longitude
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Horizontal_position_representation&amp;action=edit&amp;section=1" title="Edit section: Latitude and longitude">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main articles:
       <a href="/wiki/Latitude" title="Latitude">
        Latitude
       </a>
       and
       <a href="/wiki/Longitude" title="Longitude">
        Longitude
       </a>
      </div>
      <p>
       The most common horizontal position representation is
       <a href="/wiki/Latitude" title="Latitude">
        latitude
       </a>
       and
       <a href="/wiki/Longitude" title="Longitude">
        longitude
       </a>
       . The parameters are intuitive and well known, and are thus suited for communicating a position to humans, e.g. using a position plot.
      </p>
      <p>
       However, latitude and longitude should be used with care in mathematical expressions (including calculations in computer programs). The main reason is the
       <a class="mw-redirect" href="/wiki/Mathematical_singularity" title="Mathematical singularity">
        singularities
       </a>
       at the
       <a href="/wiki/Geographical_pole" title="Geographical pole">
        Poles
       </a>
       , which makes longitude undefined at these points. Also near the poles the latitude/longitude grid is highly non-linear, and several errors may occur in calculations that are sufficiently accurate on other locations.
       <sup class="reference" id="cite_ref-1">
        <a href="#cite_note-1">
         [1]
        </a>
       </sup>
      </p>
      <p>
       Another problematic area is the
       <a href="/wiki/Meridian_(geography)" title="Meridian (geography)">
        meridian
       </a>
       at ±
       <a href="/wiki/180th_meridian" title="180th meridian">
        180°
       </a>
       longitude, where the longitude has a
       <a class="mw-redirect" href="/wiki/Discontinuity_(mathematics)" title="Discontinuity (mathematics)">
        discontinuity
       </a>
       , and hence specific program code must often be written to handle this. An example of the consequences of omitting such code is the crash of the navigation systems of twelve
       <a class="mw-redirect" href="/wiki/F-22_Raptor" title="F-22 Raptor">
        F-22 Raptors
       </a>
       while crossing this meridian.
       <sup class="reference" id="cite_ref-2">
        <a href="#cite_note-2">
         [2]
        </a>
       </sup>
      </p>
      <h2>
       <span class="mw-headline" id="n-vector">
        <i>
         n
        </i>
        -vector
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Horizontal_position_representation&amp;action=edit&amp;section=2" title="Edit section: n-vector">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/N-vector" title="N-vector">
        n-vector
       </a>
      </div>
      <p>
       <a href="/wiki/N-vector" title="N-vector">
        <i>
         n
        </i>
        -vector
       </a>
       is a three parameter
       <a class="mw-redirect" href="/wiki/Mathematical_singularity" title="Mathematical singularity">
        non-singular
       </a>
       horizontal position representation that can replace latitude and longitude. Geometrically, it is a
       <a href="/wiki/Unit_vector" title="Unit vector">
        unit vector
       </a>
       which is normal to the
       <a href="/wiki/Reference_ellipsoid" title="Reference ellipsoid">
        reference ellipsoid
       </a>
       . The vector is
       <a href="/wiki/Vector_decomposition" title="Vector decomposition">
        decomposed
       </a>
       in an
       <a href="/wiki/ECEF" title="ECEF">
        Earth centered earth fixed
       </a>
       <a href="/wiki/Coordinate_system" title="Coordinate system">
        coordinate system
       </a>
       . It behaves the same at all Earth positions, and it holds the mathematical
       <a href="/wiki/Injective_function" title="Injective function">
        one-to-one
       </a>
       property. The vector formulation makes it possible to use standard 3D
       <a href="/wiki/Vector_calculus" title="Vector calculus">
        vector algebra
       </a>
       , and thus
       <i>
        n
       </i>
       -vector is well-suited for mathematical calculations, e.g. adding, subtracting, interpolating and averaging positions.
      </p>
      <p>
       Using three parameters,
       <i>
        n
       </i>
       -vector is inconvenient for communicating a position directly to humans and before showing a position plot, a
       <a href="/wiki/N-vector#Converting_n-vector_to_latitude/longitude" title="N-vector">
        conversion to latitude/longitude
       </a>
       might be needed.
      </p>
      <h2>
       <span class="mw-headline" id="Local_flat_Earth_assumption">
        Local flat Earth assumption
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Horizontal_position_representation&amp;action=edit&amp;section=3" title="Edit section: Local flat Earth assumption">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       When carrying out several calculations within a limited area, a
       <a href="/wiki/Cartesian_coordinate_system" title="Cartesian coordinate system">
        Cartesian coordinate system
       </a>
       might be defined with the
       <a href="/wiki/Origin_(mathematics)" title="Origin (mathematics)">
        origin
       </a>
       at a specified Earth-fixed position. The origin is often selected at the surface of the reference ellipsoid, with the
       <i>
        z
       </i>
       -axis in the vertical direction. Hence (three dimensional) position vectors relative to this coordinate frame will have two horizontal and one vertical parameter. The axes are typically selected as
       <a class="mw-redirect" href="/wiki/North_East_Down" title="North East Down">
        North-East-Down
       </a>
       or
       <a class="mw-redirect" href="/wiki/East_North_Up" title="East North Up">
        East-North-Up
       </a>
       , and thus this system can be viewed as a
       <a href="/wiki/Linearization" title="Linearization">
        linearization
       </a>
       of the
       <a href="/wiki/Meridian_(geography)" title="Meridian (geography)">
        meridians
       </a>
       and
       <a href="/wiki/Circle_of_latitude" title="Circle of latitude">
        parallels
       </a>
       .
      </p>
      <p>
       For small areas a local coordinate system can be convenient for relative positioning, but with increasing (horizontal) distances, errors will increase and repositioning of the
       <a class="mw-redirect" href="/wiki/Tangent_point" title="Tangent point">
        tangent point
       </a>
       may be required. The alignment along the north and east directions is not possible at the
       <a href="/wiki/Geographical_pole" title="Geographical pole">
        Poles
       </a>
       , and near the Poles these directions might have significant errors (here the linearization is valid only in a very small area).
      </p>
      <h2>
       <span class="mw-headline" id="UTM">
        UTM
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Horizontal_position_representation&amp;action=edit&amp;section=4" title="Edit section: UTM">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/Universal_Transverse_Mercator_coordinate_system" title="Universal Transverse Mercator coordinate system">
        Universal Transverse Mercator coordinate system
       </a>
      </div>
      <p>
       Instead of one local Cartesian grid, that needs to be repositioned as the position of interest moves, a fixed set of
       <a class="mw-redirect" href="/wiki/Map_projections" title="Map projections">
        map projections
       </a>
       covering the Earth can be defined.
       <a href="/wiki/Universal_Transverse_Mercator_coordinate_system" title="Universal Transverse Mercator coordinate system">
        UTM
       </a>
       is one such system, dividing the Earth into 60 longitude zones (and with
       <a class="mw-redirect" href="/wiki/Universal_Polar_Stereographic_coordinate_system" title="Universal Polar Stereographic coordinate system">
        UPS
       </a>
       covering the
       <a class="mw-redirect" href="/wiki/Polar_regions" title="Polar regions">
        Polar regions
       </a>
       ).
      </p>
      <p>
       UTM is widely used, and the coordinates approximately corresponds to meters north and east. However, as a set of map-projections it has inherent
       <a class="mw-redirect" href="/wiki/Distortions" title="Distortions">
        distortions
       </a>
       , and thus most calculations based on UTM will not be exact. The crossing of zones gives additional complexity.
      </p>
      <h2>
       <span class="mw-headline" id="Comparison">
        Comparison
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Horizontal_position_representation&amp;action=edit&amp;section=5" title="Edit section: Comparison">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       When deciding which parameters to use for representing position in a specific application, there are several properties that should be considered. The following table gives a summary of what to consider.
      </p>
      <table class="wikitable">
       <caption>
        Comparison of horizontal position representations
       </caption>
       <tr>
        <th>
         Representation
        </th>
        <th>
         Pros
        </th>
        <th>
         Cons
        </th>
       </tr>
       <tr>
        <td>
         <a href="/wiki/Latitude" title="Latitude">
          Latitude
         </a>
         and
         <a href="/wiki/Longitude" title="Longitude">
          longitude
         </a>
        </td>
        <td>
         <ul>
          <li>
           Widely used
          </li>
          <li>
           Parameters are easy to recognize by humans (well-suited for plotting)
          </li>
         </ul>
        </td>
        <td>
         <ul>
          <li>
           <a class="mw-redirect" href="/wiki/Mathematical_singularity" title="Mathematical singularity">
            Singularities
           </a>
           at the
           <a href="/wiki/Geographical_pole" title="Geographical pole">
            Poles
           </a>
          </li>
          <li>
           Complex behavior near the Poles
          </li>
          <li>
           Discontinuity at the ±
           <a href="/wiki/180th_meridian" title="180th meridian">
            180° meridian
           </a>
          </li>
         </ul>
        </td>
       </tr>
       <tr>
        <td>
         <a href="/wiki/N-vector" title="N-vector">
          <i>
           n
          </i>
          -vector
         </a>
        </td>
        <td>
         <ul>
          <li>
           <a class="mw-redirect" href="/wiki/Mathematical_singularity" title="Mathematical singularity">
            Nonsingular
           </a>
          </li>
          <li>
           Efficient in equations/calculations since standard 3D
           <a href="/wiki/Vector_calculus" title="Vector calculus">
            vector algebra
           </a>
           can be used
          </li>
          <li>
           All Earth positions are treated equally
          </li>
         </ul>
        </td>
        <td>
         <ul>
          <li>
           Inconvenient for communicating a position to humans
          </li>
          <li>
           Uses three parameters
          </li>
         </ul>
        </td>
       </tr>
       <tr>
        <td>
         Local
         <a href="/wiki/Cartesian_coordinate_system" title="Cartesian coordinate system">
          Cartesian coordinate system
         </a>
        </td>
        <td>
         <ul>
          <li>
           Cartesian vectors in meters along the directions of north, east and down are obtained
          </li>
         </ul>
        </td>
        <td>
         <ul>
          <li>
           Can only be used for relative positioning (the tangent point must be represented by some other quantity)
          </li>
          <li>
           Errors increase with increasing horizontal distance from the tangent point (which may require repositioning of the tangent point)
          </li>
          <li>
           North and east directions are undefined at the Poles, and near the Poles these directions may change significantly within the area of interest
          </li>
         </ul>
        </td>
       </tr>
       <tr>
        <td>
         <a href="/wiki/Universal_Transverse_Mercator_coordinate_system" title="Universal Transverse Mercator coordinate system">
          UTM
         </a>
        </td>
        <td>
         <ul>
          <li>
           Widely used
          </li>
          <li>
           Approximate north and east directions
          </li>
          <li>
           One unit corresponds approximately to one meter
          </li>
         </ul>
        </td>
        <td>
         <ul>
          <li>
           Inherent
           <a href="/wiki/Distortion" title="Distortion">
            distortion
           </a>
           (due to the
           <a href="/wiki/Map_projection" title="Map projection">
            map projection
           </a>
           ) gives only approximate answers for most calculations
          </li>
          <li>
           Calculations get complex when crossing the zones
          </li>
          <li>
           The
           <a class="mw-redirect" href="/wiki/Polar_Regions" title="Polar Regions">
            Polar Regions
           </a>
           are not covered
          </li>
         </ul>
        </td>
       </tr>
      </table>
