      <p>
       <b>
        Specials
       </b>
       is a short
       <a href="/wiki/Unicode" title="Unicode">
        Unicode
       </a>
       block allocated at the very end of the
       <a class="mw-redirect" href="/wiki/Basic_Multilingual_Plane" title="Basic Multilingual Plane">
        Basic Multilingual Plane
       </a>
       , at U+FFF0–FFFF. Of these 16 code points, five are assigned as of Unicode 10.0:
      </p>
      <ul>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFF9
         </span>
        </span>
        <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">
         INTERLINEAR ANNOTATION ANCHOR
        </span>
        , marks start of
        <a href="/wiki/Interlinear_gloss" title="Interlinear gloss">
         annotated text
        </a>
       </li>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFFA
         </span>
        </span>
        <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">
         INTERLINEAR ANNOTATION SEPARATOR
        </span>
        , marks start of annotating character(s)
       </li>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFFB
         </span>
        </span>
        <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">
         INTERLINEAR ANNOTATION TERMINATOR
        </span>
        , marks end of annotation block
       </li>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFFC
         </span>
        </span>
        <span style="font-size:125%">
         ￼
        </span>
        <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">
         OBJECT REPLACEMENT CHARACTER
        </span>
        , placeholder in the text for another unspecified object, for example in a
        <a href="/wiki/Compound_document" title="Compound document">
         compound document
        </a>
        .
       </li>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFFD
         </span>
        </span>
        <span style="font-size:125%">
         �
        </span>
        <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">
         REPLACEMENT CHARACTER
        </span>
        used to replace an unknown, unrecognized or unrepresentable character
       </li>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFFE
         </span>
        </span>
        <span style="font-size:small">
         &lt;noncharacter-FFFE&gt;
        </span>
        not a character.
       </li>
       <li>
        <span class="nowrap">
         <span class="monospaced" style="font-family: monospace, monospace;">
          U+FFFF
         </span>
        </span>
        <span style="font-size:small">
         &lt;noncharacter-FFFF&gt;
        </span>
        not a character.
       </li>
      </ul>
      <p>
       FFFE and FFFF are not unassigned in the usual sense, but
       <a href="/wiki/Universal_Character_Set_characters#Non-characters" title="Universal Character Set characters">
        guaranteed not to be a Unicode character at all
       </a>
       . They can be used to guess a text's encoding scheme, since any text containing these is by definition not a correctly encoded Unicode text. Unicode's
       <span class="nowrap">
        <span class="monospaced" style="font-family: monospace, monospace;">
         U+FEFF
        </span>
       </span>
       <span class="smallcaps" style="font-variant: small-caps; font-size: smaller;">
        <a href="/wiki/Byte_order_mark" title="Byte order mark">
         Byte order mark
        </a>
       </span>
       character can be inserted at the beginning of a Unicode text to signal its
       <a href="/wiki/Endianness" title="Endianness">
        endianness
       </a>
       : a program reading such a text and encountering 0xFFFE would then know that it should switch the byte order for all the following characters.
      </p>

      <h2>
       <span class="mw-headline" id="Replacement_character">
        Replacement character
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Specials_(Unicode_block)&amp;action=edit&amp;section=1" title="Edit section: Replacement character">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="thumb tright">
       <div class="thumbinner" style="width:222px;">
        <a class="image" href="/wiki/File:Replacement_character.svg">
         <img alt="" class="thumbimage" data-file-height="200" data-file-width="200" height="220" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/7f/Replacement_character.svg/220px-Replacement_character.svg.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/7f/Replacement_character.svg/330px-Replacement_character.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/7/7f/Replacement_character.svg/440px-Replacement_character.svg.png 2x" width="220"/>
        </a>
        <div class="thumbcaption">
         <div class="magnify">
          <a class="internal" href="/wiki/File:Replacement_character.svg" title="Enlarge">
          </a>
         </div>
         Replacement character
        </div>
       </div>
      </div>
      <p>
       The
       <b>
        replacement character
       </b>
       � (often a black diamond with a white question mark or an empty square box) is a symbol found in the
       <a href="/wiki/Unicode" title="Unicode">
        Unicode
       </a>
       standard at codepoint U+FFFD in the
       <i>
        Specials
       </i>
       table. It is used to indicate problems when a system is unable to render a stream of data to a correct symbol. It is usually seen when the data is invalid and does not match any character:
      </p>
      <p>
       Consider a
       <a href="/wiki/Text_file" title="Text file">
        text file
       </a>
       containing the German word "für" in the
       <a class="mw-redirect" href="/wiki/ISO-8859-1" title="ISO-8859-1">
        ISO-8859-1
       </a>
       encoding (
       <code>
        0x66 0xFC 0x72
       </code>
       ). This file is now opened with a text editor that assumes the input is
       <a href="/wiki/UTF-8" title="UTF-8">
        UTF-8
       </a>
       . The first and last byte are valid UTF-8 encodings of ASCII, but the middle byte (
       <code>
        0xFC
       </code>
       ) is not a valid byte in UTF-8. Therefore, a text editor could replace this byte with the replacement character symbol to produce a valid string of Unicode code points. The whole string now displays like this: "f�r".
      </p>
      <p>
       A poorly implemented text editor might save the replacement in UTF-8 form; the text file data will then look like this:
       <code>
        0x66 0xEF 0xBF 0xBD 0x72
       </code>
       , which will be displayed in ISO-8859-1 as "fï¿½r" (see
       <a href="/wiki/Mojibake" title="Mojibake">
        mojibake
       </a>
       ). Since the replacement is the same for all errors this makes it impossible to recover the original character. A better (but harder to implement) design is to preserve the original bytes, including the error, and only convert to the replacement when
       <i>
        displaying
       </i>
       the text. This will allow the text editor to save the original byte sequence, while still showing the error indicator to the user.
      </p>
      <p>
       It has become increasingly common for software to interpret invalid UTF-8 by guessing the bytes are in another byte-based encoding such as ISO-8859-1. This allows correct display of both valid and invalid UTF-8 pasted together. If a web page uses ISO-8859-1 (or
       <a href="/wiki/Windows-1252" title="Windows-1252">
        Windows-1252
       </a>
       ) but specifies the encoding as UTF-8, most web browsers used to display all non-ASCII characters as �, but newer browsers translate the erroneous bytes individually to characters in Windows-1252, so the replacement character is less frequently seen.
      </p>
