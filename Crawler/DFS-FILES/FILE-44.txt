      <p>
       In
       <a href="/wiki/Navigation" title="Navigation">
        navigation
       </a>
       ,
       <b>
        dead reckoning
       </b>
       is the process of calculating one's current position by using a previously determined position, or
       <a href="/wiki/Fix_(position)" title="Fix (position)">
        fix
       </a>
       , and advancing that position based upon known or estimated speeds over elapsed time and course. The corresponding term in biology, used to describe the processes by which animals update their estimates of position or heading, is
       <a href="/wiki/Path_integration" title="Path integration">
        path integration
       </a>
       .
      </p>
      <div class="thumb tright">
       <div class="thumbinner" style="width:222px;">
        <a class="image" href="/wiki/File:Wind_drift.png">
         <img alt="" class="thumbimage" data-file-height="1000" data-file-width="684" height="322" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/8b/Wind_drift.png/220px-Wind_drift.png" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/8b/Wind_drift.png/330px-Wind_drift.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/8b/Wind_drift.png/440px-Wind_drift.png 2x" width="220"/>
        </a>
        <div class="thumbcaption">
         <div class="magnify">
          <a class="internal" href="/wiki/File:Wind_drift.png" title="Enlarge">
          </a>
         </div>
         Drift is the angle between the heading of the airplane and the desired track. A is the last known position (fix, usually shown with a circle). B is the air position (usually shown with a plus sign). C is the DR position (usually shown with a triangle).
        </div>
       </div>
      </div>
      <p>
       Dead reckoning is subject to cumulative errors. Advances in
       <a href="/wiki/Navigational_aid" title="Navigational aid">
        navigational aids
       </a>
       that give accurate information on position, in particular
       <a href="/wiki/Satellite_navigation" title="Satellite navigation">
        satellite navigation
       </a>
       using the
       <a href="/wiki/Global_Positioning_System" title="Global Positioning System">
        Global Positioning System
       </a>
       , have made simple dead reckoning by humans obsolete for most purposes. However,
       <a href="/wiki/Inertial_navigation_system" title="Inertial navigation system">
        inertial navigation systems
       </a>
       , which provide very accurate directional information, use dead reckoning and are very widely applied.
      </p>
      <p>
       By analogy with their navigational use, the words
       <i>
        dead reckoning
       </i>
       are also used to mean the process of estimating the value of any variable quantity by using an earlier value and adding whatever changes have occurred in the meantime. Often, this usage implies that the changes are not known accurately. The earlier value and the changes may be measured or calculated quantities.
      </p>
      <p>
       The term "dead reckoning" came from "deductive" reckoning or reasoning. The use of the word "dead" to replace ded may have been a slang expression with a double entendre, since navigating in this manner for all but short distances will often lead to gross errors. For all intents and purposes "ded" is no longer used and dead is the accepted terminology.

           citation needed
          </span>
         </a>
        </i>
        ]
       </sup>
      </p>

      <h2>
       <span class="mw-headline" id="Errors">
        Errors
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=1" title="Edit section: Errors">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       Dead reckoning can give the best available information on position, but is subject to significant errors as both speed and direction must be accurately known at all instants for position to be determined accurately. For example, if displacement is measured by the number of rotations of a wheel, any discrepancy between the actual and assumed travelled distance per rotation, due perhaps to slippage or surface irregularities, will be a source of error. As each estimate of position is relative to the previous one, errors are
       <a href="/wiki/Propagation_of_uncertainty" title="Propagation of uncertainty">
        cumulative
       </a>
       , or compounding, multiplicatively or exponentially, if that is the co-relationship of the quanta.
      </p>
      <p>
       The accuracy of dead reckoning can be increased significantly by using other, more reliable methods to get a new fix part way through the journey. For example, if one was navigating on land in poor visibility, then dead reckoning could be used to get close enough to the known position of a landmark to be able to see it, before walking to the landmark itself — giving a precisely known start point — and then setting off again.
      </p>
      <h2>
       <span class="mw-headline" id="Localization_of_mobile_sensor_nodes">
        Localization of mobile sensor nodes
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=2" title="Edit section: Localization of mobile sensor nodes">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       <a href="/wiki/Location_estimation_in_sensor_networks" title="Location estimation in sensor networks">
        Localizing
       </a>
       a static
       <a href="/wiki/Sensor_node" title="Sensor node">
        sensor node
       </a>
       is not a difficult task because attaching a
       <a href="/wiki/Global_Positioning_System" title="Global Positioning System">
        GPS
       </a>
       device suffices the need of localization. But a
       <a href="/wiki/Mobile_wireless_sensor_network" title="Mobile wireless sensor network">
        mobile sensor node
       </a>
       , which continuously change its geographical location with time is difficult to localize. Mostly mobile sensor nodes within some particular domain for data collection can be used,
       <i>
        i.e
       </i>
       , sensor node attached to an animal within a grazing field or attached to a soldier on a battlefield. Within these scenarios a GPS device for each sensor node cannot be afforded. Some of the reasons for this include cost, size and battery drainage of constrained sensor nodes. To overcome this problem a limited number of reference nodes (with GPS) within a field is employed. These nodes continuously broadcast their locations and other nodes in proximity receive these locations and calculate their position using some mathematical technique like
       <a href="/wiki/Trilateration" title="Trilateration">
        trilateration
       </a>
       . For localization, at least three known reference locations are necessary to localize. Several localization algorithms based on Sequential Monte Carlo (SMC) method have been proposed in literatures.
       <sup class="reference" id="cite_ref-1">
        <a href="#cite_note-1">
         [1]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-2">
        <a href="#cite_note-2">
         [2]
        </a>
       </sup>
       Sometimes a node at some places receives only two known locations and hence it becomes impossible to localize. To overcome this problem, dead reckoning technique is used. With this technique a sensor node uses its previous calculated location for localization at later time intervals.
       <sup class="reference" id="cite_ref-3">
        <a href="#cite_note-3">
         [3]
        </a>
       </sup>
       For example, at time instant 1 if node A calculates its position as
       <i>
        loca_1
       </i>
       with the help of three known reference locations; then at time instant 2 it uses
       <i>
        loca_1
       </i>
       along with two other reference locations received from other two reference nodes. This not only localizes a node in less time but also localizes in positions where it is difficult to get three reference locations.
       <sup class="reference" id="cite_ref-4">
        <a href="#cite_note-4">
         [4]
        </a>
       </sup>
      </p>
      <h2>
       <span class="mw-headline" id="Animal_navigation">
        Animal navigation
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=3" title="Edit section: Animal navigation">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/Animal_navigation" title="Animal navigation">
        Animal navigation
       </a>
      </div>
      <p>
       In studies of
       <a href="/wiki/Animal_navigation" title="Animal navigation">
        animal navigation
       </a>
       , dead reckoning is more commonly (though not exclusively) known as
       <a href="/wiki/Path_integration" title="Path integration">
        path integration
       </a>
       . Animals use it to estimate their current location based on their movements from their last known location. Animals such as ants, rodents, and geese have been shown to track their locations continuously relative to a starting point and to return to it, an important skill for foragers with a fixed home.
       <sup class="reference" id="cite_ref-5">
        <a href="#cite_note-5">
         [5]
        </a>
       </sup>
       <sup class="reference" id="cite_ref-6">
        <a href="#cite_note-6">
         [6]
        </a>
       </sup>
      </p>
      <h2>
       <span class="mw-headline" id="Marine_navigation">
        Marine navigation
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=4" title="Edit section: Marine navigation">
         edit
        </a>
        <span class="mw-editsection-bracket">

       </span>
      </h2>
      <div class="thumb tright">
       <div class="thumbinner" style="width:222px;">
        <a class="image" href="/wiki/File:CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg">
         <img alt="" class="thumbimage" data-file-height="3240" data-file-width="4320" height="165" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/26/CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg/220px-CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/26/CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg/330px-CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/26/CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg/440px-CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg 2x" width="220"/>
        </a>
        <div class="thumbcaption">
         <div class="magnify">
          <a class="internal" href="/wiki/File:CYA_Coastal_Navigation_With_Dead_Reckoning_2.jpg" title="Enlarge">
          </a>
         </div>
         Dead reckoning navigation tools in coastal navigation
        </div>
       </div>
      </div>
      <p>
       In marine navigation a "dead" reckoning plot generally does not take into account the effect of
       <a href="/wiki/Ocean_current" title="Ocean current">
        currents
       </a>
       or
       <a href="/wiki/Wind" title="Wind">
        wind
       </a>
       . Aboard ship a dead reckoning plot is considered important in evaluating position information and planning the movement of the vessel.
       <sup class="reference" id="cite_ref-7">
        <a href="#cite_note-7">
         [7]
        </a>
       </sup>
      </p>
      <p>
       Dead reckoning begins with a known position, or
       <a href="/wiki/Fix_(position)" title="Fix (position)">
        fix
       </a>
       , which is then advanced, mathematically or directly on the chart, by means of recorded heading, speed, and time. Speed can be determined by many methods. Before modern instrumentation, it was determined aboard ship using a
       <a href="/wiki/Chip_log" title="Chip log">
        chip log
       </a>
       . More modern methods include
       <a class="mw-redirect" href="/wiki/Pit_log" title="Pit log">
        pit log
       </a>
       referencing engine speed (
       <i>
        e.g
       </i>
       . in
       <a href="/wiki/Revolutions_per_minute" title="Revolutions per minute">
        rpm
       </a>
       ) against a table of total displacement (for ships) or referencing one's indicated airspeed fed by the pressure from a
       <a href="/wiki/Pitot_tube" title="Pitot tube">
        pitot tube
       </a>
       . This measurement is converted to an
       <a href="/wiki/Equivalent_airspeed" title="Equivalent airspeed">
        equivalent airspeed
       </a>
       based upon known atmospheric conditions and measured errors in the indicated airspeed system. A naval vessel uses a device called a
       <a href="/wiki/Pit_sword" title="Pit sword">
        pit sword
       </a>
       (rodmeter), which uses two sensors on a metal rod to measure the electromagnetic variance caused by the ship moving through water. This change is then converted to ship's speed. Distance is determined by multiplying the speed and the time. This initial position can then be adjusted resulting in an estimated position by taking into account the current (known as
       <a href="/wiki/Set_and_drift" title="Set and drift">
        set and drift
       </a>
       in marine navigation). If there is no positional information available, a new dead reckoning plot may start from an estimated position. In this case subsequent dead reckoning positions will have taken into account estimated set and drift.
      </p>
      <p>
       Dead reckoning positions are calculated at predetermined intervals, and are maintained between fixes. The duration of the interval varies. Factors including one's speed made good and the nature of heading and other course changes, and the navigator's judgment determine when dead reckoning positions are calculated.
      </p>
      <p>
       Before the 18th-century development of the
       <a href="/wiki/Marine_chronometer" title="Marine chronometer">
        marine chronometer
       </a>
       by
       <a href="/wiki/John_Harrison" title="John Harrison">
        John Harrison
       </a>
       and the
       <a href="/wiki/Lunar_distance_(navigation)" title="Lunar distance (navigation)">
        lunar distance method
       </a>
       , dead reckoning was the primary method of determining
       <a href="/wiki/Longitude" title="Longitude">
        longitude
       </a>
       available to mariners such as
       <a href="/wiki/Christopher_Columbus" title="Christopher Columbus">
        Christopher Columbus
       </a>
       and
       <a href="/wiki/John_Cabot" title="John Cabot">
        John Cabot
       </a>
       on their trans-Atlantic voyages. Tools such as the
       <a href="/wiki/Traverse_board" title="Traverse board">
        traverse board
       </a>
       were developed to enable even illiterate crew members to collect the data needed for dead reckoning.
       <a href="/wiki/Polynesian_navigation" title="Polynesian navigation">
        Polynesian navigation
       </a>
       , however, uses different
       <a href="/wiki/Wayfinding" title="Wayfinding">
        wayfinding
       </a>
       techniques.
      </p>
      <h2>
       <span class="mw-headline" id="Air_navigation">
        Air navigation
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=5" title="Edit section: Air navigation">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/Air_navigation" title="Air navigation">
        Air navigation
       </a>
      </div>
      <div class="thumb tright">
       <div class="thumbinner" style="width:222px;">
        <a class="image" href="/wiki/File:RCAF_DR_Nav_Tools_1940.JPG">
         <img alt="" class="thumbimage" data-file-height="3240" data-file-width="4320" height="165" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a9/RCAF_DR_Nav_Tools_1940.JPG/220px-RCAF_DR_Nav_Tools_1940.JPG" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a9/RCAF_DR_Nav_Tools_1940.JPG/330px-RCAF_DR_Nav_Tools_1940.JPG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a9/RCAF_DR_Nav_Tools_1940.JPG/440px-RCAF_DR_Nav_Tools_1940.JPG 2x" width="220"/>
        </a>
        <div class="thumbcaption">
         <div class="magnify">
          <a class="internal" href="/wiki/File:RCAF_DR_Nav_Tools_1940.JPG" title="Enlarge">
          </a>
         </div>
         British P10 Magnetic Compass with dead reckoning navigation tools
        </div>
       </div>
      </div>
      <p>
       On May 21, 1927
       <a href="/wiki/Charles_Lindbergh" title="Charles Lindbergh">
        Charles Lindbergh
       </a>
       landed in
       <a class="mw-redirect" href="/wiki/Paris,_France" title="Paris, France">
        Paris, France
       </a>
       after a successful non-stop flight from the United States in the single-engined
       <a href="/wiki/Spirit_of_St._Louis" title="Spirit of St. Louis">
        Spirit of St. Louis
       </a>
       . This aircraft was equipped with very basic instruments. He used dead reckoning to find his way.
      </p>
      <p>
       Dead reckoning in the air is similar to dead reckoning on the sea, but slightly more complicated. The density of the air the aircraft moves through affects its performance as well as winds, weight, and power settings.
      </p>
      <p>
       The basic formula for DR is Distance = Speed x Time. An aircraft flying at 250 knots airspeed for 2 hours has flown 500 nautical miles through the air. The
       <a href="/wiki/Wind_triangle" title="Wind triangle">
        wind triangle
       </a>
       is used to calculate the effects of wind on heading and airspeed to obtain a magnetic heading to steer and the speed over the ground (groundspeed). Printed tables, formulae, or an
       <a href="/wiki/E6B" title="E6B">
        E6B
       </a>
       flight computer are used to calculate the effects of air density on aircraft rate of climb, rate of fuel burn, and airspeed.
       <sup class="reference" id="cite_ref-8">
        <a href="#cite_note-8">
         [8]
        </a>
       </sup>
      </p>
      <p>
       A course line is drawn on the aeronautical chart along with estimated positions at fixed intervals (say every ½ hour). Visual observations of ground features are used to obtain fixes. By comparing the fix and the estimated position corrections are made to the aircraft's heading and groundspeed.
      </p>
      <p>
       Dead reckoning is on the curriculum for VFR (visual flight rules - or basic level) pilots worldwide.
       <sup class="reference" id="cite_ref-9">
        <a href="#cite_note-9">
         [9]
        </a>
       </sup>
       It is taught regardless of whether the aircraft has navigation aids such as GPS,
       <a class="mw-redirect" href="/wiki/Automatic_direction_finder" title="Automatic direction finder">
        ADF
       </a>
       and
       <a href="/wiki/VHF_omnidirectional_range" title="VHF omnidirectional range">
        VOR
       </a>
       and is an
       <a class="mw-redirect" href="/wiki/ICAO" title="ICAO">
        ICAO
       </a>
       Requirement. Many flying training schools will prevent a student from using electronic aids until they have mastered dead reckoning.
      </p>
      <p>
       <a href="/wiki/Inertial_navigation_system" title="Inertial navigation system">
        Inertial navigation systems
       </a>
       (INSes), which are nearly universal on more advanced aircraft, use dead reckoning internally. The INS provides reliable navigation capability under virtually any conditions, without the need for external navigation references, although it is still prone to slight errors.
      </p>
      <h2>
       <span class="mw-headline" id="Automotive_navigation">
        <span id="ADR">
        </span>
        Automotive navigation
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=6" title="Edit section: Automotive navigation">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/Automatic_vehicle_location" title="Automatic vehicle location">
        Automatic vehicle location
       </a>
      </div>
      <p>
       Dead reckoning is today implemented in some high-end
       <a href="/wiki/Automotive_navigation_system" title="Automotive navigation system">
        automotive navigation systems
       </a>
       in order to overcome the limitations of
       <a class="mw-redirect" href="/wiki/GPS" title="GPS">
        GPS
       </a>
       /
       <a class="mw-redirect" href="/wiki/GNSS" title="GNSS">
        GNSS
       </a>
       technology alone. Satellite microwave signals are unavailable in
       <a class="mw-redirect" href="/wiki/Parking_garage" title="Parking garage">
        parking garages
       </a>
       and tunnels, and often severely degraded in
       <a class="mw-redirect" href="/wiki/Urban_canyon" title="Urban canyon">
        urban canyons
       </a>
       and near trees due to blocked lines of sight to the satellites or
       <a href="/wiki/Multipath_propagation" title="Multipath propagation">
        multipath propagation
       </a>
       . In a dead-reckoning navigation system, the car is equipped with sensors that know the wheel circumference and record wheel rotations and steering direction. These sensors are often already present in cars for other purposes (
       <a href="/wiki/Anti-lock_braking_system" title="Anti-lock braking system">
        anti-lock braking system
       </a>
       ,
       <a href="/wiki/Electronic_stability_control" title="Electronic stability control">
        electronic stability control
       </a>
       ) and can be read by the navigation system from the
       <a class="mw-redirect" href="/wiki/Controller-area_network" title="Controller-area network">
        controller-area network
       </a>
       bus. The navigation system then uses a
       <a href="/wiki/Kalman_filter" title="Kalman filter">
        Kalman filter
       </a>
       to integrate the always-available sensor data with the accurate but occasionally unavailable position information from the satellite data into a combined position fix.
      </p>
      <h2>
       <span class="mw-headline" id="Autonomous_navigation_in_robotics">
        Autonomous navigation in robotics
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=7" title="Edit section: Autonomous navigation in robotics">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       Dead reckoning is utilized in some lower-end, non mission-critical, or tightly constrained by time or weight, robotic applications. It is usually used to reduce the need for sensing technology, such as
       <a class="mw-redirect" href="/wiki/Ultrasonic_sensor" title="Ultrasonic sensor">
        ultrasonic sensors
       </a>
       , GPS, or placement of some
       <a href="/wiki/Linear_encoder" title="Linear encoder">
        linear
       </a>
       and
       <a href="/wiki/Rotary_encoder" title="Rotary encoder">
        rotary encoders
       </a>
       , in an
       <a href="/wiki/Autonomous_robot" title="Autonomous robot">
        autonomous robot
       </a>
       , thus greatly reducing cost and complexity at the expense of performance and repeatability. The proper utilization of dead reckoning in this sense would be to supply a known percentage of electrical power or
       <a href="/wiki/Hydraulics" title="Hydraulics">
        hydraulic
       </a>
       pressure to the robot's drive motors over a given amount of time from a general starting point. Dead reckoning is not totally accurate, which can lead to errors in distance estimates ranging from a few millimeters (in
       <a href="/wiki/Numerical_control" title="Numerical control">
        CNC machining
       </a>
       ) to kilometers (in
       <a href="/wiki/Unmanned_aerial_vehicle" title="Unmanned aerial vehicle">
        UAVs
       </a>
       ), based upon the duration of the run, the speed of the robot, the length of the run, and several other factors.
      </p>
      <h2>
       <span class="mw-headline" id="Pedestrian_dead_reckoning">
        <span id="PDR">
        </span>
        <span id="UDR">
        </span>
        Pedestrian dead reckoning
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=8" title="Edit section: Pedestrian dead reckoning">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       With the increased sensor offering in
       <a href="/wiki/Smartphone" title="Smartphone">
        smartphones
       </a>
       , built-in accelerometers can be used as a
       <a href="/wiki/Pedometer" title="Pedometer">
        pedometer
       </a>
       and built-in magnetometer as a compass heading provider.
       <b>
        Pedestrian dead reckoning
       </b>
       (
       <b>
        PDR
       </b>
       ) can be used to supplement other navigation methods in a similar way to automotive navigation, or to extend navigation into areas where other navigation systems are unavailable.
       <sup class="reference" id="cite_ref-10">
        <a href="#cite_note-10">
         [10]
        </a>
       </sup>
      </p>
      <p>
       In a simple implementation, the user holds their phone in front of them and each step causes position to move forward a fixed distance in the direction measured by the compass. Accuracy is limited by the sensor precision, magnetic disturbances inside structures, and unknown variables such as carrying position and stride length. Another challenge is differentiating walking from running, and recognizing movements like bicycling, climbing stairs, or riding an elevator.
      </p>
      <p>
       Before phone-based systems existed, many custom PDR systems existed. While a
       <a href="/wiki/Pedometer" title="Pedometer">
        pedometer
       </a>
       can only be used to measure linear distance traveled, PDR systems have an embedded magnetometer for heading measurement. Custom PDR systems can take many forms including special boots, belts, and watches, where the variability of carrying position has been minimized to better utilize magnetometer heading. True dead reckoning is fairly complicated, as it is not only important to minimize basic drift, but also to handle different carrying scenarios and movements, as well as hardware differences across phone models.
      </p>
      <h2>
       <span class="mw-headline" id="Directional_dead_reckoning">
        Directional dead reckoning
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=9" title="Edit section: Directional dead reckoning">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <div class="hatnote navigation-not-searchable" role="note">
       Main article:
       <a href="/wiki/South-pointing_chariot" title="South-pointing chariot">
        South-pointing chariot
       </a>
      </div>
      <p>
       The south-pointing chariot was an ancient Chinese device consisting of a two-wheeled
       <a href="/wiki/Horse-drawn_vehicle" title="Horse-drawn vehicle">
        horse-drawn vehicle
       </a>
       which carried a pointer that was intended always to aim to the south, no matter how the chariot turned. The chariot pre-dated the navigational use of the
       <a href="/wiki/Compass#Magnetic_compass" title="Compass">
        magnetic compass
       </a>
       , and could not
       <i>
        detect
       </i>
       the direction that was south. Instead it used a kind of
       <b>
        directional dead reckoning
       </b>
       : at the start of a journey, the pointer was aimed southward by hand, using local knowledge or astronomical observations e.g. of the
       <a class="mw-redirect" href="/wiki/Pole_Star" title="Pole Star">
        Pole Star
       </a>
       . Then, as it traveled, a mechanism possibly containing
       <a href="/wiki/Differential_(mechanical_device)" title="Differential (mechanical device)">
        differential
       </a>
       gears used the different rotational speeds of the two wheels to turn the pointer relative to the body of the chariot by the angle of turns made (subject to available mechanical accuracy), keeping the pointer aiming in its original direction, to the south. Errors, as always with dead reckoning, would accumulate as distance traveled increased.
      </p>
      <h2>
       <span class="mw-headline" id="For_networked_games">
        For networked games
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [
        </span>
        <a href="/w/index.php?title=Dead_reckoning&amp;action=edit&amp;section=10" title="Edit section: For networked games">
         edit
        </a>
        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>

       <sup class="reference" id="cite_ref-Murphy2011_11-0">
        <a href="#cite_note-Murphy2011-11">

       <sup class="reference" id="cite_ref-12">

         [12]
        </a>
       </sup>
      </p>

      <h2>
       <span class="mw-headline" id="Computer_science">
        Computer science
       </span>
       <span class="mw-editsection">
        <span class="mw-editsection-bracket">
         [

         edit

        <span class="mw-editsection-bracket">
         ]
        </span>
       </span>
      </h2>
      <p>
       In computer science, dead-reckoning refers to navigating an
       <a href="/wiki/Array_data_structure" title="Array data structure">
        array data structure
       </a>
       using indexes. Since every element's array has the same size, it is possible to
       <a href="/wiki/Random_access" title="Random access">
        directly access
       </a>
       one array element by knowing any position in the array.
       <sup class="reference" id="cite_ref-14">
        <a href="#cite_note-14">
         [14]
        </a>
       </sup>
      </p>
      <p>
       Given the following array:
      </p>

        <td>
         E
        </td>

      </table>

       knowing the memory address where the array starts, it is easy to compute the memory address of
       <tt>
        D
       </tt>
